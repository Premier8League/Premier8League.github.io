Logistic Regression

Owing to its simplicity and parallelization, logistic regression has become the go-to tool for many machine learning tasks. By exploring from mathematical dimension, this post helps you to gain a better understanding of the algorithm.

Published

11 August 2020

At the early age, logistic regression maintatines its stronghold as the most effective method in both CTR (Click-Through Rate) and CVR (Conversion Rate) prediction, where we estimate the probability of a user’s click or conversion on a given item. Though it has many more complex extensions, here we focus on the its naïve form, i.e. modeling the probabilities for classification problems with two possible outcomes.

Here I would like to explain the maths behind logistic regression, how we use maximum likelihood estimation and how we fit the model.

Odds and Logit

Before we start dissecting the algorithm closely, let us first review two metrics. In statistic, the odds of an event is the ratio of the probability that the event will happen to the probability that the event will not happen. For instance, suppose the chances of a user’s conversion on his clicked ads is P(conversion∣clicked)=pP(\text{conversion}\mid\text{clicked})=pP(conversion∣clicked)=p, and thus the chances of no conversion is P(no conversion∣clicked)=1−pP(\text{no conversion}\mid\text{clicked})=1-pP(no conversion∣clicked)=1−p. Therefore, the odds can be expressed as:

odds=p1−p \text{odds}=\frac{p}{1-p} odds=1−pp​

Logit, or log-odds, is a function that maps probability from [0,1][0,1][0,1] to (−∞,∞)(-\infty,\infty)(−∞,∞). Given a probability ppp, the logit of the probability is

logit⁡(p)=log⁡(p1−p) \operatorname{logit}(p)=\log \left(\frac{p}{1-p}\right) logit(p)=log(1−pp​)

One of the merits of logit is that, odds are multiplied or divided frequently, and logarithm converts multiplication to addition and division to subtractions. This is quite computational friendly. You may know that the last layer of neural networks is called logit layer, which will return the raw values ranging from (−∞,∞)(-\infty,\infty)(−∞,∞) for the predictions.

Logistic Regression

The main use-case of a logistic model is to be given a feature input x=(x1,x2,…,xm)\boldsymbol{x} =(x_1,x_2,\dots,x_m)x=(x1​,x2​,…,xm​) , and estimate the probability ppp that an event y=1y=1y=1. By defination, we have:

p=11+e−w⊺x p = \frac{1}{1+e^{-\boldsymbol{w^\intercal x}}} p=1+e−w⊺x1​

where w\boldsymbol{w}w are the parameters of the model. Interestingly, after some transformation, we can have a more intuitive picture:

p=11+e−w⊺x  ⟹  1p=1+e−w⊺x  ⟹  1−pp=1ew⊺x  ⟹  p1−p=ew⊺x  ⟹  log⁡(p1−p)=w⊺x \begin{aligned} p = \frac{1}{1+e^{-\boldsymbol{w^\intercal x}}} \implies & \frac{1}{p}= 1+ e^{-\boldsymbol{w^\intercal x}} \\[2ex] \implies &\frac{1-p}{p} = \frac{1}{e^{\boldsymbol{w^\intercal x}}}\\[2ex] \implies &\frac{p}{1-p} = e^{\boldsymbol{w^\intercal x}}\\[2ex] \implies &\log \left(\frac{p}{1-p}\right) = \boldsymbol{w^\intercal x}\\ \end{aligned} p=1+e−w⊺x1​⟹⟹⟹⟹​p1​=1+e−w⊺xp1−p​=ew⊺x1​1−pp​=ew⊺xlog(1−pp​)=w⊺x​

The above formula shows that, the core of logistic regression lies in the linear approximation between the feature inputs and the logit of the event that y=1y=1y=1 w.r.t. parameters w\boldsymbol{w}w.

The goal of the logistic regression is to model the probability of a random variable yyy being 000 or 111 given experimental data. Specifically:

P⁡(y=1∣x;w)=hw(x)=11+e−w⊺xP⁡(y=0∣x;w)=1−hw(x) \begin{aligned} \operatorname{P}(y=1 \mid \boldsymbol{x} ; \boldsymbol{w}) &= h_{\boldsymbol{w}}(\boldsymbol{x})=\frac{1}{1+e^{-\boldsymbol{w^\intercal x}}}\\ \operatorname{P}(y=0 \mid \boldsymbol{x} ; \boldsymbol{w}) &= 1- h_{\boldsymbol{w}}(\boldsymbol{x}) \end{aligned} P(y=1∣x;w)P(y=0∣x;w)​=hw​(x)=1+e−w⊺x1​=1−hw​(x)​

Since y∈{0,1}y\in\{0,1\}y∈{0,1}, it can be written more compactly as:

P⁡(y∣x;w)=hw(x)y(1−hw(x))(1−y) \operatorname{P}(y \mid \boldsymbol{x} ; \boldsymbol{w})= h_{\boldsymbol{w}}(\boldsymbol{x})^y(1- h_{\boldsymbol{w}}(\boldsymbol{x}))^{(1-y)} P(y∣x;w)=hw​(x)y(1−hw​(x))(1−y)

What Is Likelihood Function?

With the given set of experimental data (X,y)={(x1,y1),(x2,y2),…,(xm,ym)}(\boldsymbol{X,y})=\{(\boldsymbol{x}^1,y^1),(\boldsymbol{x}^2,y^2),\dots,(\boldsymbol{x}^m,y^m)\}(X,y)={(x1,y1),(x2,y2),…,(xm,ym)}, our next step is to find the parameter values for which the probability of the observed data

P(y∣X;w)P(\boldsymbol{y}\mid\boldsymbol{X};\boldsymbol{w})P(y∣X;w)

is greatest. Note that xi\boldsymbol{x}^ixi represents the ithi^{th}ith observation, and totally we have mmm observations. Since each yiy^iyi represents a binomial count, i.e. each observation is independent, the joint probability density function of y\boldsymbol{y}y can be written as:

P(y∣X;w)=∏i=1mP(yi∣xi;w)=∏i=1mhw(xi)yi(1−hw(xi))(1−yi)(1) \begin{aligned} P(\boldsymbol{y}\mid\boldsymbol{X};\boldsymbol{w}) &= \prod_{i=1}^m P(y^i\mid \boldsymbol{x}^i;\boldsymbol{w})\\ &= \prod_{i=1}^m h_{\boldsymbol{w}}(\boldsymbol{x}^i)^{y^i}(1- h_{\boldsymbol{w}}(\boldsymbol{x}^i))^{(1-y^i)}\\ \tag{1} \end{aligned} P(y∣X;w)​=i=1∏m​P(yi∣xi;w)=i=1∏m​hw​(xi)yi(1−hw​(xi))(1−yi)​(1)

Eq. (1)(1)(1) is viewed as a function of y\boldsymbol{y}y with fixed w\boldsymbol{w}w. When we wish to explicitly view it as a function of w\boldsymbol{w}w (since we want to find the best parameters), we will instead call it the likelihood function:

L(w)=L(w∣X,y)=∏i=1mP(yi∣xi;w)=∏i=1mhw(xi)yi(1−hw(xi))(1−yi) \begin{aligned} L(\boldsymbol{w}) = L(\boldsymbol{w}\mid \boldsymbol{X,y}) &= \prod_{i=1}^m P(y^i\mid \boldsymbol{x}^i;\boldsymbol{w})\\ &= \prod_{i=1}^m h_{\boldsymbol{w}}(\boldsymbol{x}^i)^{y^i}(1- h_{\boldsymbol{w}}(\boldsymbol{x}^i))^{(1-y^i)}\\ \end{aligned} L(w)=L(w∣X,y)​=i=1∏m​P(yi∣xi;w)=i=1∏m​hw​(xi)yi(1−hw​(xi))(1−yi)​

How Does Maximum Likelihood Estimation Work?

Literally, what maximum likelihood estimation (MLE) does is to maximize the likelihood function L(w)L(\boldsymbol{w})L(w). However, it is cumbersome to differentiate such quantity and can be simplified a great deal further by taking its log:

J(w)=log⁡L(w)=log⁡∏i=1mhw(xi)yi(1−hw(xi))(1−yi)=∑i=1myilog⁡hw(xi)+(1−yi)log⁡(1−hw(xi))(2) \begin{aligned} J(\boldsymbol{w}) &= \log L(\boldsymbol{w})\\ & = \log \prod_{i=1}^m h_{\boldsymbol{w}}(\boldsymbol{x}^i)^{y^i}(1- h_{\boldsymbol{w}}(\boldsymbol{x}^i))^{(1-y^i)}\\ &= \sum_{i=1}^m y^i\log h_{\boldsymbol{w}}(\boldsymbol{x}^i)+(1-y^i)\log (1- h_{\boldsymbol{w}}(\boldsymbol{x}^i))\\ \tag{2} \end{aligned} J(w)​=logL(w)=logi=1∏m​hw​(xi)yi(1−hw​(xi))(1−yi)=i=1∑m​yiloghw​(xi)+(1−yi)log(1−hw​(xi))​(2)

The critical points of J(w)J(\boldsymbol{w})J(w) occur when the first derivative w.r.t. each wkw_kwk​ equals 000. Note that since the other terms in the summation do not depend on wkw_kwk​ and can thus be treated as constants, we have:

∂w⊺xi∂wk=∂∂wk(∑k=0Kxkiwk)=xki \frac{\partial \boldsymbol{w^\intercal x^i}}{\partial w_k} = \frac{\partial}{\partial w_k}(\sum_{k=0}^K x_k^iw_k) = x_k^i ∂wk​∂w⊺xi​=∂wk​∂​(k=0∑K​xki​wk​)=xki​

Furthermore, the derivative of sigmoid function w.r.t. wkw_kwk​ is:

∂hw(xi)∂wk=∂∂wk11+e−w⊺xi=−e−w⊺xi(1+e−w⊺xi)2⋅∂w⊺xi∂wk=hw(xi)(1−hw(xi))xki \begin{aligned} \frac{\partial h_{\boldsymbol{w}}(\boldsymbol{x}^i)}{\partial w_k}& = \frac{\partial}{\partial w_k}\frac{1}{1+e^{-\boldsymbol{w^\intercal x^i}}} \\[2ex] &= \frac{-e^{-\boldsymbol{w^\intercal x^i}}}{(1+e^{-\boldsymbol{w^\intercal x^i}})^2}\cdot\frac{\partial \boldsymbol{w^\intercal x^i}}{\partial w_k}\\[2ex] &= h_{\boldsymbol{w}}(\boldsymbol{x}^i)(1-h_{\boldsymbol{w}}(\boldsymbol{x}^i))x^i_k \end{aligned} ∂wk​∂hw​(xi)​​=∂wk​∂​1+e−w⊺xi1​=(1+e−w⊺xi)2−e−w⊺xi​⋅∂wk​∂w⊺xi​=hw​(xi)(1−hw​(xi))xki​​

Thereby, differentiating Eq.(2)(2)(2) w.r.t. each wkw_kwk​ we have:

∂J(w)∂wk=∑i=1myi∂log⁡hw(xi)∂wk+(1−yi)∂log⁡(1−hw(xi))∂wk=∑i=1myi1hw(xi)∂hw(xi)∂wk+(1−yi)1(1−hw(xi))∂(1−hw(xi))∂wk=∑i=1m(yi−hw(xi))xki \begin{aligned} \frac{\partial J(\boldsymbol{w})}{\partial w_k} &= \sum_{i=1}^m y^i\frac{\partial\log h_{\boldsymbol{w}}(\boldsymbol{x}^i)}{\partial w_k}+(1-y^i)\frac{\partial \log (1- h_{\boldsymbol{w}}(\boldsymbol{x}^i))}{\partial w_k}\\[2ex] &= \sum_{i=1}^m y^i\frac{1}{h_{\boldsymbol{w}}(\boldsymbol{x}^i)}\frac{\partial h_{\boldsymbol{w}}(\boldsymbol{x}^i)}{\partial w_k}+(1-y^i)\frac{1}{(1- h_{\boldsymbol{w}}(\boldsymbol{x}^i))}\frac{\partial (1- h_{\boldsymbol{w}}(\boldsymbol{x}^i))}{\partial w_k}\\[2ex] &=\sum_{i=1}^m (y^i-h_{\boldsymbol{w}}(\boldsymbol{x}^i))x^i_k \end{aligned} ∂wk​∂J(w)​​=i=1∑m​yi∂wk​∂loghw​(xi)​+(1−yi)∂wk​∂log(1−hw​(xi))​=i=1∑m​yihw​(xi)1​∂wk​∂hw​(xi)​+(1−yi)(1−hw​(xi))1​∂wk​∂(1−hw​(xi))​=i=1∑m​(yi−hw​(xi))xki​​

With gradient ascent, we can maximize the likelihood iteratively by

where α\alphaα is the learning rate.

We conclude this post with two extra points:

  • If we consider the output of the logistic regression as the generated probability distribution and the target labels as the expected probability distribution, then mathematically, MLE is asymptotically equivalent to minimizing Kullback–Leibler divergence and cross entropy of these two distributions. Check this for the proof.

  • Some may wonder why should we use maximum likelihood estimation but not least squares method (LSM) as it is used for linear regression? The quick answer is that MLE with negative log-likelihood is a convex optimization problem, where every local minimum is a global minimum, while LSM is a non-convex optimization problem and multiple local minimums may exist. Check this for the proof.
